## 提交代码
1. git status查看差异文件
2. 交互选择需提交的文件
3. 根据提交文件的功能，补全提交记录描述
4. 推送到github

##  提交记录描述样板

```
# <类型>: <简短描述>
# 
# <详细说明>
# 
# <关闭/关联的 Issue>
# 
# --- 提交类型 ---
# feat:     新功能
# fix:      修复 Bug
# docs:     文档变更
# style:    代码格式（不影响功能，如空格、分号等）
# refactor: 重构（既不是新功能也不是修复 Bug）
# perf:     性能优化
# test:     添加或修改测试
# chore:    构建过程或辅助工具变更
# 
# --- 格式说明 ---
# 1. 标题行不超过 50 个字符
# 2. 标题首字母大写，末尾不加句号
# 3. 标题与详细说明之间空一行
# 4. 详细说明每行不超过 72 个字符
# 5. 使用现在时态（"add" 而不是 "added"）
# 
# --- 示例 ---
# [feat]: 添加用户登录功能
# 
# 实现了基于 JWT 的用户认证
# - 添加登录 API 接口
# - 添加 Token 验证中间件
# - 更新用户模型
# 
# Closes #123
```
